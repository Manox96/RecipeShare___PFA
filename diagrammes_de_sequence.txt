DIAGRAMMES DE SÉQUENCE - RECIPE SHARE
=====================================

📋 DIAGRAMMES DE SÉQUENCE PRINCIPAUX
====================================

🔹 1. INSCRIPTION D'UN NOUVEL UTILISATEUR
=========================================

TITRE: Inscription d'un nouvel utilisateur
ACTEURS: Utilisateur Invité, Page d'inscription, Système, Base de Données

SÉQUENCE:
1. Utilisateur Invité → Page d'inscription
2. Utilisateur Invité → Saisit nom d'utilisateur, email, mot de passe
3. Utilisateur Invité → Clic sur "S'inscrire"
4. Page d'inscription → Validation des données
5. Page d'inscription → Vérification unicité username/email
6. Page d'inscription → Base de Données (créer utilisateur)
7. Base de Données → Confirmation création
8. Page d'inscription → Création profil automatique
9. Page d'inscription → Connexion automatique
10. Page d'inscription → Redirection vers page principale
11. Système → Affichage message de succès

🔹 2. CONNEXION UTILISATEUR
===========================

TITRE: Connexion d'un utilisateur
ACTEURS: Utilisateur Invité, Page de connexion, Système, Base de Données

SÉQUENCE:
1. Utilisateur Invité → Page de connexion
2. Utilisateur Invité → Saisit username et mot de passe
3. Utilisateur Invité → Clic sur "Se connecter"
4. Page de connexion → Validation des champs
5. Page de connexion → Base de Données (vérifier identifiants)
6. Base de Données → Retourne données utilisateur
7. Page de connexion → Vérification mot de passe
8. Page de connexion → Création session utilisateur
9. Page de connexion → Redirection vers page principale
10. Système → Affichage message de bienvenue

🔹 3. CRÉATION D'UNE RECETTE
============================

TITRE: Création d'une nouvelle recette
ACTEURS: Utilisateur Inscrit, Page de création, Système, Base de Données

SÉQUENCE:
1. Utilisateur Inscrit → Page de création de recette
2. Utilisateur Inscrit → Saisit titre, description, difficulté
3. Utilisateur Inscrit → Sélectionne cuisine et type de repas
4. Utilisateur Inscrit → Saisit temps de préparation et cuisson
5. Utilisateur Inscrit → Saisit nombre de portions
6. Utilisateur Inscrit → Ajoute des ingrédients (nom, quantité, unité)
7. Utilisateur Inscrit → Ajoute des étapes de préparation
8. Utilisateur Inscrit → Sélectionne des étiquettes
9. Utilisateur Inscrit → Télécharge une image (optionnel)
10. Utilisateur Inscrit → Définit la visibilité (public/privé)
11. Utilisateur Inscrit → Clic sur "Créer la recette"
12. Page de création → Validation des données
13. Page de création → Base de Données (sauvegarder recette)
14. Base de Données → Confirmation création
15. Page de création → Redirection vers détail de la recette
16. Système → Affichage message de succès

🔹 4. AJOUT AUX FAVORIS
=======================

TITRE: Ajout d'une recette aux favoris
ACTEURS: Utilisateur Inscrit, Page de détail recette, Système, Base de Données

SÉQUENCE:
1. Utilisateur Inscrit → Page de détail d'une recette
2. Utilisateur Inscrit → Clic sur bouton "Ajouter aux favoris"
3. Page de détail → Vérification si déjà en favoris
4. Page de détail → Base de Données (créer relation favorite)
5. Base de Données → Confirmation création
6. Page de détail → Mise à jour interface (bouton devient "Retirer")
7. Système → Affichage message "Ajouté aux favoris"

🔹 5. AJOUT D'UN COMMENTAIRE
============================

TITRE: Ajout d'un commentaire sur une recette
ACTEURS: Utilisateur Inscrit, Page de détail recette, Système, Base de Données

SÉQUENCE:
1. Utilisateur Inscrit → Page de détail d'une recette
2. Utilisateur Inscrit → Saisit commentaire dans le formulaire
3. Utilisateur Inscrit → Clic sur "Publier le commentaire"
4. Page de détail → Validation du commentaire
5. Page de détail → Base de Données (sauvegarder commentaire)
6. Base de Données → Confirmation création
7. Page de détail → Actualisation de la liste des commentaires
8. Système → Affichage du nouveau commentaire
9. Système → Affichage message de succès

🔹 6. GESTION DES UTILISATEURS (ADMIN)
======================================

TITRE: Gestion des utilisateurs par l'administrateur
ACTEURS: Administrateur, Dashboard Admin, Système, Base de Données

SÉQUENCE:
1. Administrateur → Dashboard administrateur
2. Administrateur → Clic sur "Gérer les utilisateurs"
3. Dashboard Admin → Base de Données (récupérer tous les utilisateurs)
4. Base de Données → Retourne liste des utilisateurs
5. Dashboard Admin → Affichage tableau des utilisateurs
6. Administrateur → Sélectionne un utilisateur
7. Administrateur → Clic sur "Modifier" ou "Supprimer"
8. Dashboard Admin → Base de Données (action demandée)
9. Base de Données → Confirmation de l'action
10. Dashboard Admin → Actualisation du tableau
11. Système → Affichage message de confirmation

🔹 8. EXPORT DES DONNÉES (ADMIN)
================================

TITRE: Export des données de recettes par l'administrateur
ACTEURS: Administrateur, Dashboard Admin, Système, Base de Données

SÉQUENCE:
1. Administrateur → Dashboard administrateur
2. Administrateur → Clic sur "Statistiques détaillées"
3. Dashboard Admin → Affichage page des statistiques
4. Administrateur → Clic sur "Exporter les recettes (CSV)"
5. Dashboard Admin → Base de Données (récupérer toutes les recettes)
6. Base de Données → Retourne données des recettes
7. Dashboard Admin → Génération du fichier CSV
8. Dashboard Admin → Téléchargement du fichier CSV
9. Système → Confirmation de téléchargement

TITRE: Recherche et filtrage de recettes publiques
ACTEURS: Utilisateur, Page principale, Système, Base de Données

1. Utilisateur → Page principale
2. Utilisateur → Sélectionne type de repas (optionnel)
3. Utilisateur → Saisit terme de recherche (optionnel)
4. Utilisateur → Clic sur "Rechercher"
5. Page principale → Base de Données (recherche avec filtres)
6. Base de Données → Retourne recettes correspondantes
7. Page principale → Affichage des résultats paginés
8. Système → Affichage nombre de résultats trouvés

7. Base de Données → Confirmation suppression
8. Page de détail → Redirection vers liste des recettes
9. Système → Affichage message de confirmation

🎨 CONVENTIONS VISUELLES
========================

ACTEURS:
- Utilisateur Invité (utilisateur non connecté)
- Utilisateur Inscrit (utilisateur connecté)
- Administrateur (utilisateur avec droits admin)
- Système (logique métier)
- Base de Données (persistance des données)

MESSAGES:
- Flèche pleine → Appel de méthode/action
- Flèche pointillée → Retour de données
- Flèche épaisse → Création d'objet

LIFELINES:
- Activation → Période d'exécution
- Destruction → Fin de vie de l'objet

FRAGMENTS:
- alt → Alternatives (si/else)
- loop → Boucles
- opt → Optionnel
- ref → Référence à un autre diagramme

💡 TIPS POUR CRÉATION VISUELLE
==============================

1. UTILISEZ DES COULEURS DIFFÉRENTES:
   - Utilisateur: Bleu
   - Système: Vert
   - Base de Données: Orange
   - Messages d'erreur: Rouge

2. AJOUTEZ DES NOTES:
   - Conditions de validation
   - Messages d'erreur possibles
   - Points de décision

3. INCLUEZ LES EXCEPTIONS:
   - Données invalides
   - Erreurs de base de données
   - Conflits d'accès

4. UTILISEZ DES FRAGMENTS:
   - Pour les validations complexes
   - Pour les boucles (ajout d'ingrédients)
   - Pour les alternatives (succès/échec)
