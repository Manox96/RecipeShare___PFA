CAS D'USAGE DÉTAILLÉS :
=======================

AUTHENTIFICATION ET GESTION UTILISATEUR :
-----------------------------------------

UC1 - S'inscrire
- Acteur principal : Utilisateur invité
- Préconditions : Aucune
- Flux principal :
  1. L'utilisateur accède à la page d'inscription
  2. L'utilisateur saisit son nom d'utilisateur, email et mot de passe
  3. Le système valide les informations
  4. Le système crée le compte utilisateur
  5. L'utilisateur est connecté automatiquement
- Postconditions : L'utilisateur a un compte actif

UC2 - Se connecter
- Acteur principal : Utilisateur invité
- Préconditions : L'utilisateur a un compte
- Flux principal :
  1. L'utilisateur accède à la page de connexion
  2. L'utilisateur saisit ses identifiants
  3. Le système valide les identifiants
  4. L'utilisateur est connecté
- Postconditions : L'utilisateur est authentifié

UC3 - Se déconnecter
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est connecté
- Flux principal :
  1. L'utilisateur clique sur "Déconnexion"
  2. Le système ferme la session
- Postconditions : L'utilisateur n'est plus connecté

GESTION DES RECETTES :
---------------------

UC4 - Parcourir les recettes publiques
- Acteur principal : Utilisateur invité, Utilisateur inscrit
- Préconditions : Aucune
- Flux principal :
  1. L'utilisateur accède à la page principale
  2. Le système affiche les recettes publiques
  3. L'utilisateur peut filtrer par type de repas
  4. L'utilisateur peut voir les recettes paginées
- Postconditions : L'utilisateur voit les recettes disponibles

UC5 - Voir les détails d'une recette
- Acteur principal : Utilisateur invité, Utilisateur inscrit
- Préconditions : La recette existe et est publique
- Flux principal :
  1. L'utilisateur clique sur une recette
  2. Le système affiche les détails complets
  3. L'utilisateur voit les ingrédients, étapes et commentaires
- Postconditions : L'utilisateur a vu tous les détails de la recette

UC6 - Créer une recette
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est connecté
- Flux principal :
  1. L'utilisateur accède au formulaire de création
  2. L'utilisateur saisit le titre, description, difficulté
  3. L'utilisateur ajoute les ingrédients et quantités
  4. L'utilisateur ajoute les étapes de préparation
  5. L'utilisateur définit la visibilité
  6. Le système sauvegarde la recette
- Postconditions : La recette est créée et visible selon les paramètres

UC7 - Modifier une recette
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est propriétaire de la recette
- Flux principal :
  1. L'utilisateur accède à la page de modification
  2. L'utilisateur modifie les informations
  3. Le système sauvegarde les modifications
- Postconditions : La recette est mise à jour

FONCTIONNALITÉS SOCIALES :
-------------------------

UC8 - Ajouter aux favoris
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est connecté, la recette existe
- Flux principal :
  1. L'utilisateur clique sur "Ajouter aux favoris"
  2. Le système ajoute la recette aux favoris
- Postconditions : La recette apparaît dans les favoris

UC9 - Voir mes favoris
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est connecté
- Flux principal :
  1. L'utilisateur accède à la page des favoris
  2. Le système affiche toutes les recettes favorites
- Postconditions : L'utilisateur voit ses recettes favorites

UC10 - Ajouter un commentaire
- Acteur principal : Utilisateur inscrit
- Préconditions : L'utilisateur est connecté, la recette existe
- Flux principal :
  1. L'utilisateur saisit son commentaire
  2. Le système sauvegarde le commentaire
- Postconditions : Le commentaire est visible sur la recette

GESTION DU CONTENU :
-------------------

UC11 - Gérer les utilisateurs
- Acteur principal : Administrateur
- Préconditions : L'utilisateur a les droits d'administration
- Flux principal :
  1. L'administrateur accède au tableau de bord
  2. L'administrateur peut voir tous les utilisateurs
  3. L'administrateur peut modifier ou supprimer des utilisateurs
- Postconditions : Les utilisateurs sont gérés selon les actions

UC12 - Gérer les étiquettes
- Acteur principal : Administrateur
- Préconditions : L'utilisateur a les droits d'administration
- Flux principal :
  1. L'administrateur accède à la gestion des étiquettes
  2. L'administrateur peut créer, modifier ou supprimer des étiquettes
- Postconditions : Les étiquettes sont mises à jour

UC13 - Gérer les ingrédients
- Acteur principal : Administrateur
- Préconditions : L'utilisateur a les droits d'administration
- Flux principal :
  1. L'administrateur accède à la gestion des ingrédients
  2. L'administrateur peut créer, modifier ou supprimer des ingrédients
- Postconditions : Les ingrédients sont mis à jour

SYSTÈME DE BLOG :
----------------

UC14 - Voir les articles de blog
- Acteur principal : Utilisateur invité, Utilisateur inscrit
- Préconditions : Aucune
- Flux principal :
  1. L'utilisateur accède à la section blog
  2. Le système affiche tous les articles
- Postconditions : L'utilisateur voit les articles disponibles

UC15 - Créer un article de blog
- Acteur principal : Administrateur
- Préconditions : L'utilisateur a les droits d'administration
- Flux principal :
  1. L'administrateur accède au formulaire de création
  2. L'administrateur saisit le titre et le contenu
  3. L'administrateur peut ajouter une image
  4. L'administrateur peut ajouter des étiquettes
  5. Le système publie l'article
- Postconditions : L'article est publié et visible

RELATIONS INCLUDE/EXTEND :
=========================

INCLUDE :
- Voir les détails d'une recette INCLUDE Parcourir les recettes
- Créer une recette INCLUDE Ajouter des ingrédients
- Créer une recette INCLUDE Ajouter des étapes
- Créer une recette INCLUDE Ajouter des étiquettes
- Modifier une recette INCLUDE Ajouter des ingrédients
- Modifier une recette INCLUDE Ajouter des étapes
- Modifier une recette INCLUDE Ajouter des étiquettes
- Créer un article de blog INCLUDE Ajouter des étiquettes

EXTEND :
- Ajouter aux favoris EXTEND Voir les détails d'une recette
- Ajouter un commentaire EXTEND Voir les détails d'une recette
- Ajouter une photo aux favoris EXTEND Voir ses photos

FREQUENCE D'UTILISATION :
========================

- Parcourir les recettes : Très fréquent (plusieurs fois par jour)
- Voir les détails d'une recette : Fréquent (quotidien)
- Créer une recette : Occasionnel (hebdomadaire)
- Ajouter aux favoris : Fréquent (quotidien)
- Gérer les utilisateurs : Rare (administrateur uniquement)
- Créer un article de blog : Occasionnel (administrateur uniquement)
- Gérer les étiquettes/ingrédients : Rare (administrateur uniquement)

EXIGENCES SPÉCIALES :
====================

- Les images doivent être au format JPG, PNG ou GIF
- Taille maximale des images : 5MB
- Le titre de la recette doit être unique par utilisateur
- Le système doit gérer la création concurrente de recettes
- Les mots de passe doivent contenir au moins 8 caractères
- Les emails doivent être uniques dans le système
- Les utilisateurs avec is_staff=True ont accès aux fonctions d'administration
